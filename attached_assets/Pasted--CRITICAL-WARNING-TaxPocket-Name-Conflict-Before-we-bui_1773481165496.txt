📌 Project Vision

SnapBill – "The Snapchat for Your Receipts and Bills." A mobile‑first web app (PWA) where users snap a photo of any receipt or bill, and it automatically saves, organizes, and manages it. Combines instant capture with smart financial tools.

🎯 Core Features to Build

1. Camera-First Landing Page

· When user opens the app, the first thing they see is the camera view (like Snapchat).
· Big shutter button at bottom center.
· Permission request for camera access (with fallback to file upload).
· Clean, minimal UI – no distractions.

2. Instant Capture & Save

· User takes photo → immediate feedback (shutter animation).
· Auto‑upload to server for processing.
· Show a quick success toast: "Receipt saved!" with optional undo.
· Store original image in Firestore (or Cloud Storage) for later reference.

3. Upload Options (Manual)

· If camera not available, user can:
  · Choose photo from gallery
  · Upload PDF
  · Drag & drop (desktop)
· Same processing pipeline as camera capture.

4. Email Forwarding (Same as MyBillPort)

· Generate unique email address per user: receipts+userid@snapbill.com
· User forwards receipts/bills from vendors to this address.
· Incoming emails are parsed using the same extraction pipeline (OCR + Claude) and added to their pending queue.

5. Automatic Data Extraction

· Use your existing extractBillDataEnhanced pipeline:
  · Google Vision OCR for text
  · Claude for structuring (vendor, amount, date, category)
  · Biller‑specific parsers for known providers
· Store extracted data in Firestore under receipts collection (or unify with bills).

6. Timeline View (Like Snapchat Stories)

· Receipts appear in a reverse‑chronological timeline.
· Group by month/year.
· Each receipt card shows: vendor, amount, date, category icon.
· Tap to expand full details and image.

7. Search & Organization

· Search by vendor, amount, date range.
· Auto‑categorize (meals, travel, office supplies, utilities).
· Tagging system (user‑defined).

8. Bill Management (Reuse MyBillPort Code)

· Import all existing bill management features:
  · Due date tracking
  · Payment reminders
  · Pay via redirect (or future card)
  · Recurring detection
  · Spike alerts

9. One‑Tap Export (Tax Ready)

· "Export Tax Bundle" button → generates PDF with all receipts for selected period, categorized by CRA rules.
· Option to share with accountant via secure link.

🧱 Technical Architecture

Frontend (Next.js + Tailwind)

· Mobile‑first PWA with camera access.
· Use react-webcam for camera capture.
· Offline support via service worker (cache receipts locally).

Backend (Firebase + Vercel API)

· Firestore collections:
  · receipts – stores extracted data and image URL
  · receiptImages (Storage) – original images
  · emailAliases – for forwarding
  · users – preferences, subscription status
· API routes:
  · /api/receipt/upload – handles image upload + extraction
  · /api/email-forward – same as MyBillPort
  · /api/receipts/query – search/filter receipts
  · /api/export/tax-bundle – generates PDF

OCR Pipeline (Reuse Existing)

· Google Vision → Claude → biller parsers → confidence scoring
· Same as MyBillPort's enhanced extraction.
