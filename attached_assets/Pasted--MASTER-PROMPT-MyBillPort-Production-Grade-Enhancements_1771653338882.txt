🧠 Detailed Requirements

1. App Check

· Use reCAPTCHA v3. Obtain site key and secret key.
· In lib/firebase.ts:
  · Initialize App Check with ReCaptchaV3Provider.
  · Enable debug token in development (check process.env.NODE_ENV).
· In pages/api/extract-bill.ts:
  · Read x-firebase-appcheck header.
  · Verify token using Firebase Admin's getAppCheck().verifyToken().
  · Return 401 if invalid/missing.
· Provide instructions to enable enforcement in Firebase Console after monitoring.

2. Abnormal Usage Logging

Client-side detection (utils/securityMonitor.ts):

· Export functions trackBillCreation(), trackFailedScan(errorType).
· Keep in‑memory arrays of timestamps, call Cloud Function reportSuspiciousActivity if thresholds exceeded.

Callable Cloud Function (reportSuspiciousActivity):

· Accept activityType, details, confidence.
· Extract userId from context.auth, IP, user agent.
· Write structured log to Cloud Logging with appropriate severity.
· Store a copy in Firestore securityReports collection.
· If confidence === 'high', trigger email alert (use existing email config).

Firestore Trigger (monitorBillCreationRate):

· On bills creation, count user's bills in last hour.
· If >20, log warning and create security report.

3. Daily Usage Analytics

Analytics Service (services/analyticsService.ts):

· Export functions: trackBillCreated, trackBillPaid, trackBillScanAttempt, trackUserLogin, trackFeatureUsed.
· Each function calls logEvent from Firebase Analytics with relevant parameters.

Integrate into pages:

· pages/add-bill.tsx – after successful bill creation, call trackBillCreated with method ('manual' | 'suggested' | 'scan').
· pages/bill/[id].tsx – after marking paid, call trackBillPaid with onTime boolean.
· components/BillScanner.tsx – on success/failure, call trackBillScanAttempt.

Daily Report Cloud Function (dailyUsageReport):

· Scheduled via Pub/Sub every day at 8 AM Toronto time.
· Query Firestore for counts: new users, new bills, bills paid, scan attempts, feedback from previous day.
· Compute scan success rate.
· Store report in dailyReports collection.
· Send email via nodemailer (using existing Gmail config) with a formatted HTML summary.
