🔐 Biometric Payments for MyBillPort – Feature Implementation Prompt

Paste this into Replit AI (or your coding assistant) to add Face ID / Biometric authentication for payments. This feature enhances security and user experience by allowing users to confirm payments with their device's biometrics (Face ID, Touch ID, fingerprint, etc.) via WebAuthn/passkeys.

---

🎯 Feature Overview

· What: A user setting to require biometric verification before processing any payment (e.g., when clicking "Pay Now" on a bill).
· Why: Stronger security (prevents unauthorized payments), faster access (no codes), and modern user expectation.
· How: Use the Web Authentication API (WebAuthn) to create and authenticate with passkeys (supported on all modern devices: macOS with Touch ID, Windows Hello, Android fingerprint, iPhone Face ID/Touch ID, etc.).

✅ Requirements

1. User Interface (Settings Page)

· Add a toggle in /settings labeled "Enable Face ID / Biometric for Payments".
· Show a brief description: "Use your device's biometrics to confirm payments quickly and securely."
· When toggled ON:
  · If the device supports WebAuthn and the user hasn't registered yet, trigger passkey creation.
  · If registration succeeds, store the credential ID in the user's Firestore profile and enable the feature.
· When toggled OFF:
  · Remove the credential from the user's profile (optional: also remove from authenticator? We can keep for future re-enablement but should delete from our DB).
· Display status: "Biometric enabled" or "Not enabled".

2. WebAuthn Registration Endpoint

· Route: POST /api/webauthn/register
· Function:
  · Generate a WebAuthn registration challenge (using a library like @simplewebauthn/server).
  · Store the challenge temporarily (in Firestore or session).
  · Return challenge and user info to the frontend.

3. Frontend Registration

· When user enables the feature, call the registration endpoint and use navigator.credentials.create() to create a passkey.
· Send the attestation response back to a verification endpoint.

4. WebAuthn Registration Verification Endpoint

· Route: POST /api/webauthn/verify-registration
· Function:
  · Verify the attestation using the stored challenge.
  · If valid, store the credential ID and public key in Firestore under users/{userId}/biometricCredential.
  · Enable the feature flag in user profile.

5. Payment Flow Integration

· When user clicks "Pay Now" on a bill:
  · Check if biometric payments are enabled for that user.
  · If yes, initiate WebAuthn authentication (get a challenge from server, call navigator.credentials.get()).
  · Send the assertion to a verification endpoint.
  · If successful, proceed with the payment action (redirect or API call).
  · If fails, show error and do not proceed.

6. WebAuthn Authentication Endpoints

· POST /api/webauthn/authenticate – returns a challenge for the user's credential.
· POST /api/webauthn/verify-authentication – verifies the assertion and returns success/failure.

7. Firestore Schema Updates

· Add a subcollection users/{userId}/biometric or a document with:
  · credentialId: string
  · publicKey: string (base64 encoded)
  · enabled: boolean
  · createdAt: timestamp

8. Fallback

· If device does not support WebAuthn, show a message: "Biometric authentication is not supported on this device."

---

🧱 Technical Implementation Details

Dependencies

· @simplewebauthn/server – for server-side WebAuthn operations.
· @simplewebauthn/browser – for client-side WebAuthn (optional; you can use native navigator.credentials directly).

Environment Variables

· No new secrets needed, but ensure the app is served over HTTPS (WebAuthn requires secure context).

Security Considerations

· Store challenge in Firestore with a TTL (e.g., 5 minutes) to prevent replay.
· Always verify the origin matches your domain.
· Use proper encoding (base64url) for binary data.
