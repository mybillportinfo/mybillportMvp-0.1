🔧 PHASE 1 — Google Cloud OAuth Setup (Exact)

Step 1.1 — Enable Gmail API
	1.	Go to Google Cloud Console
	2.	Select your MyBillPort project
	3.	APIs & Services → Library
	4.	Search for Gmail API → Click Enable

⸻

Step 1.2 — Create OAuth 2.0 Credentials
	1.	APIs & Services → Credentials
	2.	Click Create Credentials → OAuth client ID
	3.	Choose Web application
	4.	Name: MyBillport Gmail Integration

Add Redirect URI (must be exact):

https://mybillport.com/api/gmail/callback

No trailing slash. No previews.

Save the Client ID + Client Secret.

⸻

Step 1.3 — OAuth Consent Screen
	1.	APIs & Services → OAuth consent screen
	2.	User Type: External
	3.	App Name: MyBillport
	4.	User Support Email: your business email
	5.	Developer contact email: same
	6.	Add domain: mybillport.com

Add Scopes

Click “Add or remove scopes,” then select:

https://www.googleapis.com/auth/gmail.readonly

Nothing else.

Add Test Users (while testing)

Add:
	•	Your Gmail
	•	Any testers you want to allow initially

Only these emails can authorize until you get verified.

📝 PHASE 2 — Update Your Privacy Policy

Google requires explicit statements about Gmail access.

Add exactly:

⸻

Gmail Integration Disclosure Example

Gmail Integration
MyBillPort may request read-only access to your Gmail account to help detect billing emails. We use this access to identify billing messages, extract billing metadata — such as biller name, amount due, and due date — and create or update bills in your account.

What we access:
Only email metadata (sender, subject, snippet) in messages that contain billing information. We do not access unrelated emails.

How we use it:
To help you automate bill tracking. Extractions are used to populate your bill dashboard.

What we store:
Only the extracted billing data (biller name, amount due, due date). We do not store full email content.

Disconnection:
You may disconnect Gmail access at any time, which revokes access tokens and deletes associated data.

Third-party sharing:
We do not share your Gmail data. We use Google API Services only as described.

🔧 PHASE 3 — Set Environment Variables in Vercel

Go to:

Vercel → Project → Settings → Environment Variables

Add exactly:

Key	Value
GOOGLE_CLIENT_ID	Your Client ID
GOOGLE_CLIENT_SECRET	Your Client Secret
GOOGLE_REDIRECT_URI	https://mybillport.com/api/gmail/callback
FIREBASE_SERVICE_ACCOUNT	Full JSON of your Firebase service account

⚠ Do not include quotes or line breaks incorrectly in JSON.

🧠 Troubleshooting

📍 Redirect mismatch

Error: redirect_uri_mismatch
Fix: Exact match in Google console + code + environment variable.

📍 Access denied

Error: access_denied
Fix: Add user as test user in OAuth consent screen.

📍 Tokens not refreshing

Fix: Ensure refresh token is saved on first consent.

📍 Firebase auth fails

Ensure API routes verify Firebase ID token from Authorization header.
