🔥 Master Prompt: Enhance Calendar with Automatic Bill Due Dates & Income Markers

Paste this into Replit AI (or your coding assistant) to upgrade the calendar feature in MyBillPort.

🎯 Objective

Enhance the existing calendar so it automatically displays:

· Bill due dates (from the bills collection) as red markers.
· Income/pay dates (from a new income collection) as green markers.
· When a user taps a day, show a summary of all bills and income on that day.

📦 Data Sources

1. Bills Collection (Already Exists)

· Each document contains: userId, billerName, amount, dueDate (Firestore Timestamp), status (paid/unpaid), etc.
· For calendar, include only unpaid bills (or optionally all, but mark paid ones differently).

2. Income Collection (New)

Create a new Firestore collection income with this structure:

```typescript
{
  userId: string,
  amount: number,
  date: Timestamp,          // the payday
  frequency: 'once' | 'weekly' | 'biweekly' | 'monthly' | 'yearly',
  description?: string,
  createdAt: Timestamp
}
```

· Users will add income entries via a simple modal on the calendar.
· Security rules: allow users to read/write only their own income.

🔧 Functional Requirements

1. Calendar Grid (Existing)

· Show a monthly grid (Sun–Sat).
· On each day cell:
  · Display a green dot if there is any income on that day.
  · Display a red dot if there is any unpaid bill due that day.
  · If both exist, show both dots (or a combined indicator).
  · Optionally show a count badge for multiple bills.

2. Income Input

· When user taps a day, open a modal (if no income for that day) allowing them to add income.
· Fields: amount, frequency (once is default), optional description.
· Save to Firestore and immediately update calendar.

3. Daily Summary Popup

· When user taps a day with events, show a modal listing:
  · Income entries (amount, description) with a delete/edit option.
  · Bills due (biller name, amount, status) with a "Mark as Paid" button (calls existing function).
  · Total income and total bills for that day.

4. Data Fetching

· On month change, query:
  · bills where userId == currentUser and dueDate is within the displayed month (using Firestore range queries).
  · income where userId == currentUser and date is within the displayed month (for non‑recurring) – handle recurring separately (see below).

5. Recurring Income (Stretch)

· For MVP, treat all income as one‑time entries (frequency = 'once').
· For future: implement recurring logic (e.g., if frequency is 'monthly', generate occurrences from the base date).

6. Persistence

· Income entries must persist in Firestore.
· When user navigates away and returns, data remains.

🧱 Implementation Plan

1. Create Income Service (services/incomeService.ts):
   · addIncome(userId, incomeData)
   · updateIncome(incomeId, updates)
   · deleteIncome(incomeId)
   · getIncomeForMonth(userId, year, month)
2. Modify Calendar Page (app/calendar/page.tsx):
   · Use useAuth to get user.
   · Fetch bills and income for current month (using useEffect with month/year dependency).
   · Store in state.
   · Pass data to calendar grid component.
3. Update Calendar Grid Component:
   · For each day, determine if it has income and/or bills.
   · Render appropriate dots/badges.
4. Create Income Modal Component (reuse existing modal style):
   · Form with amount, date (pre‑filled from selected day), frequency dropdown, description.
   · On submit, call addIncome.
5. Update Day Popup Component:
   · Display lists of bills and income for that day.
   · Include "Mark as Paid" for bills (using existing function).
   · Include edit/delete for income.
6. Firestore Security Rules:
   ```
   match /income/{doc} {
     allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
     allow create: if request.auth != null && request.auth.uid == request.resource.data.userId;
   }
   ```

✅ Acceptance Criteria

· Income added on a date persists after navigation.
· Bills due dates show on calendar as red markers.
· Income dates show as green markers.
· Tapping a day with events shows a summary with bills and income.
· "Mark as Paid" updates bill status and removes it from calendar (or marks as paid).
· Income can be edited/deleted.
· Calendar navigation (previous/next month) works.
