🧩 DETECTION ALGORITHM SPECIFICATION

Recurring Detection

Trigger: Every time a bill is created/updated.

Logic:

1. Query all bills from same user + same billerName.
2. Sort by dueDate ascending.
3. If count >= 2, calculate intervals between consecutive due dates.
4. Identify dominant frequency:
   · If intervals average 28–31 days → monthly
   · If intervals average 85–95 days → quarterly
   · If intervals average 360–370 days → yearly
5. Confidence score = (number of consistent intervals / total intervals) * (min(occurrences, 5)/5)
6. If confidence >= 0.7 AND frequency detected → set recurring = true, store frequency.
7. If recurring already true but frequency changed → update and alert.

Amount Change Detection

Trigger: Bill creation/update (only if recurring = true).

Logic:

1. Calculate average amount of last 3 bills (or all if less) for same biller.
2. Compare current amount:
   · If |current - avg| > avg * 0.15 OR |current - avg| > 10 → significant change.
3. Create alert if not already dismissed for this bill/cycle.
4. Store amountChangePercent and averageAmount on the bill.
